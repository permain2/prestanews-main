1. SEO Guide

Full SEO guide available in: SEO-GUIDE.md

Key Points Summary

  • Title Tags: 50-60 characters, primary keyword near start
  • Meta Descriptions: 150-160 characters, include CTA
  • H1 Tags: One per page, matches content intent
  • Content Length: 1,500-3,000 words for articles
  • Internal Linking: 3-5 internal links per article
  • Images: Alt text required, WebP format preferred

Article Publishing Checklist

Pre-Writing:
- [ ] Keyword research completed
- [ ] Competitor top 3 articles analyzed
- [ ] Content outline with H2/H3 structure

Content Creation:
- [ ] Title 50-60 chars with primary keyword
- [ ] Meta description 150-160 chars
- [ ] Hero image optimized (WebP, alt text)
- [ ] 3-5 internal links added
- [ ] Author assigned with bio link

Technical:
- [ ] Schema markup verified
- [ ] Mobile preview checked
- [ ] Page speed acceptable

2. Writing Bot & Vocabulary System

How It Works

Use AI (Claude/GPT) with specific vocabulary and style guidelines to generate consistent, on-brand content that matches The Points Guy style.

Vocabulary Guidelines

Use ThisInstead Of
premium travel rewardsfancy credit card benefits
welcome bonussign-up offer
annual feeyearly cost
earning ratehow much you get back
redemption valuehow much points are worth
lounge accessairport lounge entry
elite statustop tier membership
transfer partnersairline/hotel partners

Creating New Review Articles

Prompt Template for AI:

"Write a credit card review for [CARD NAME] in the style of 
The Points Guy / NerdWallet. Include:

1. Card Overview (2-3 paragraphs)
2. Key Features (welcome bonus, earning rates, annual fee)
3. Pros list (4-6 items)
4. Cons list (2-4 items)
5. Who Should Apply section
6. Redemption Value Analysis
7. Bottom Line recommendation

Use professional, authoritative tone. Include specific 
numbers and percentages. Mention competitor comparisons 
where relevant.

Output in Markdown format with YAML frontmatter for Astro."

Review File Template

6. Email Opt-ins System

Full documentation in: NEWSLETTER-SETUP.md

How It Works

  1. User enters email in newsletter form (footer, sidebar, hero)
  2. Form submits to /api/subscribe endpoint
  3. API validates email and checks for duplicates
  4. Stores in Upstash Redis with metadata
  5. Returns success/error response

API Endpoints

EndpointMethodPurpose
/api/subscribePOSTAdd new subscriber
/api/subscribersGETList all subscribers (admin)

Data Structure in Redis

Key: subscriber:{email}
Value: {
  "email": "user@example.com",
  "source": "footer",
  "subscribedAt": "2025-12-10T10:30:00.000Z",
  "ip": "..."
}

Exporting Subscribers

# JSON format
curl "https://prestanews.com/api/subscribers?key=YOUR_ADMIN_KEY"

# CSV format (for import to email tools)
curl "https://prestanews.com/api/subscribers?key=YOUR_ADMIN_KEY&format=csv"

Sending Emails with Cursor

Since subscribers are stored in Upstash, you can:

  1. Export CSV from the API endpoint
  2. Import to Mailchimp, ConvertKit, Beehiiv, etc.
  3. Send campaigns through those platforms

7. Backlink Strategy

Domain Purchase Rationale

prestanews.com was purchased because:

  • Existing domain authority from previous site
  • Backlinks already pointing to it
  • Relevant to finance/news niche
  • Clean history (no spam penalties)

Link Juice Optimization Actions

  • Set up proper 301 redirects from old URLs to new structure
  • Maintain URL structure where possible for existing backlinks
  • Implement canonical tags on all pages
  • Submit XML sitemap to Google Search Console
  • Audit existing backlinks with Ahrefs/SEMrush
  • Reach out to high-value backlink sources for updates
  • Build new backlinks through guest posting

Internal Linking Strategy

  • Every article links to 3-5 related articles
  • Category pages link to all relevant sub-pages
  • Footer contains links to all main sections
  • Breadcrumbs on all pages for navigation + SEO

Tools for Backlink Analysis

  • Ahrefs: Comprehensive backlink database
  • SEMrush: Competitor analysis
  • Google Search Console: Free, shows linking domains
  • Moz Link Explorer: Domain authority tracking
